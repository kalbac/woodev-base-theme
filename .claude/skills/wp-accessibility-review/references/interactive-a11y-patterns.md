# Interactive Accessibility Patterns

Use this reference for modals, menus, tabs, accordions, and JS-heavy interactions.

## Review For

- Keyboard access to all controls
- Focus moved appropriately into overlays
- Focus returned after close
- Expanded/collapsed state reflected accessibly
- Async updates announced when needed

## Common Problems

- Escape key does nothing
- Tab can move behind modal
- State changes only indicated by color or icon
- Hidden content remains keyboard-focusable
